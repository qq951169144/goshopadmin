# 服务端项目结构说明

## 目录结构

```
backend/
├── api/              # API路由定义
│   └── routes.go     # 路由注册文件
├── config/           # 配置管理

│   └── .env.example  # 环境变量示例
├── controllers/      # 控制器
│   ├── auth.go       # 认证相关控制器
│   ├── merchant.go   # 商户相关控制器
│   ├── product.go    # 商品相关控制器
│   ├── order.go      # 订单相关控制器
│   └── activity.go   # 活动相关控制器
├── middleware/       # 中间件
│   ├── auth.go       # 认证中间件
│   ├── cors.go       # CORS中间件
│   └── logger.go     # 日志中间件
├── models/           # 数据模型
│   ├── user.go       # 用户模型
│   ├── role.go       # 角色模型
│   ├── product.go    # 商品模型
│   ├── order.go      # 订单模型
│   └── activity.go   # 活动模型
├── services/         # 业务逻辑
│   ├── auth.go       # 认证服务
│   ├── merchant.go   # 商户服务
│   ├── product.go    # 商品服务
│   ├── order.go      # 订单服务
│   └── activity.go   # 活动服务
├── utils/            # 工具函数
│   ├── jwt.go        # JWT工具
│   ├── password.go   # 密码处理
│   ├── validator.go  # 数据验证
│   └── response.go   # 响应处理
├── main.go           # 服务端入口
├── go.mod            # Go模块文件
└── go.sum            # 依赖校验文件
```

## 目录说明

### api/
- **用途**: 定义所有API路由
- **routes.go**: 集中注册所有控制器路由，实现路由与控制器的映射

### config/
- **用途**: 管理应用配置
- **config.go**: 定义配置结构，实现配置加载逻辑
- **.env.example**: 提供环境变量配置示例

### controllers/
- **用途**: 处理HTTP请求，调用相应的服务层方法
- **auth.go**: 处理登录、权限验证等认证相关请求
- **merchant.go**: 处理商户入驻、信息管理等请求
- **product.go**: 处理商品发布、管理等请求
- **order.go**: 处理订单创建、查询等请求
- **activity.go**: 处理活动配置、管理等请求

### middleware/
- **用途**: 实现HTTP中间件
- **auth.go**: 认证中间件，验证请求的JWT token
- **cors.go**: CORS中间件，处理跨域请求
- **logger.go**: 日志中间件，记录请求日志

### models/
- **用途**: 定义数据模型，实现数据库操作
- **user.go**: 用户模型，对应数据库用户表
- **role.go**: 角色模型，对应数据库角色表
- **product.go**: 商品模型，对应数据库商品表
- **order.go**: 订单模型，对应数据库订单表
- **activity.go**: 活动模型，对应数据库活动表

### services/
- **用途**: 实现业务逻辑
- **auth.go**: 认证服务，处理登录、权限验证等业务逻辑
- **merchant.go**: 商户服务，处理商户相关业务逻辑
- **product.go**: 商品服务，处理商品相关业务逻辑
- **order.go**: 订单服务，处理订单相关业务逻辑
- **activity.go**: 活动服务，处理活动相关业务逻辑

### utils/
- **用途**: 提供通用工具函数
- **jwt.go**: JWT相关工具，生成和验证token
- **password.go**: 密码处理工具，加密和验证密码
- **validator.go**: 数据验证工具，验证请求数据
- **response.go**: 响应处理工具，统一API响应格式

### main.go
- **用途**: 服务端程序入口
- **功能**: 初始化配置、设置路由、启动HTTP服务器

## 代码规范

1. **命名规范**:
   - 包名: 小写字母，使用简短的名称
   - 结构体: 大驼峰命名法
   - 函数: 大驼峰命名法
   - 变量: 小驼峰命名法
   - 常量: 全大写，下划线分隔

2. **代码风格**:
   - 使用Go 1.20+语法
   - 缩进: 使用Tab（4个空格）
   - 每行长度: 不超过100个字符
   - 函数长度: 单个函数不超过50行

3. **错误处理**:
   - 明确返回错误
   - 错误信息要清晰明确
   - 避免使用panic

4. **注释规范**:
   - 包、函数、结构体都要有注释
   - 注释要清晰、准确
   - 使用GoDoc风格注释

## 开发流程

1. **添加新功能**:
   - 在models/中定义数据模型
   - 在services/中实现业务逻辑
   - 在controllers/中处理HTTP请求
   - 在api/routes.go中注册路由

2. **修改配置**:
   - 在config/config.go中添加配置项
   - 在.env.example中添加环境变量示例
   - 在部署环境中设置相应的环境变量

3. **测试**:
   - 为每个服务层函数编写单元测试
   - 使用Postman或curl测试API接口
   - 确保所有测试通过后再提交代码

## 部署说明

1. **环境要求**:
   - Go 1.20+
   - MySQL 8.0+
   - Redis 7.0+
   - RabbitMQ 3.10+

2. **构建步骤**:
   ```bash
   cd backend
   go build -o goshopadmin-server .
   ```

3. **运行步骤**:
   - 设置环境变量
   - 启动服务: `./goshopadmin-server`

4. **Docker部署**:
   - 使用Dockerfile构建镜像
   - 使用Docker Compose编排服务

## 注意事项

1. **安全性**:
   - 密码必须加密存储
   - JWT密钥必须保密
   - 防止SQL注入和XSS攻击

2. **性能**:
   - 使用Redis缓存热点数据
   - 优化数据库查询
   - 合理使用goroutine处理并发请求

3. **可维护性**:
   - 遵循代码规范
   - 保持代码结构清晰
   - 编写详细的注释

4. **扩展性**:
   - 采用分层架构
   - 依赖注入
   - 接口定义清晰
