## 二、环境运行逻辑

### 1. 开发环境

#### 1.1 后端服务
- **运行方式**：使用 `golang:1.24-alpine` 镜像
- **代码同步**：本地 `backend` 目录挂载到容器 `/app` 目录
- **运行命令**：`go run main.go`
- **热更新**：本地修改代码后，需要重启后端容器以加载新代码
- **访问地址**：`http://localhost:8080`

#### 1.2 前端服务
- **运行方式**：使用 `node:20-alpine` 镜像
- **代码同步**：本地 `frontend` 目录挂载到容器 `/app` 目录
- **运行命令**：根据 `NODE_ENV` 环境变量自动选择
  - 开发环境：`npm run dev -- --host`
  - 生产环境：`npm run build`
- **热更新**：Vite开发服务器支持热更新，若热更新不生效，需要重启前端容器
- **访问地址**：`http://localhost:3000` (开发环境)
- **构建产物**：生产环境构建的 `dist` 文件存储在 `frontend/dist` 目录

#### 1.3 数据库与中间件
- **MySQL**：运行在 `mysql:8.0` 镜像，数据持久化到本地 `docker/data/mysql` 目录
- **Redis**：运行在 `redis:7.0` 镜像，数据持久化到本地 `docker/data/redis` 目录
- **RabbitMQ**：运行在 `rabbitmq:3.10-management` 镜像，数据持久化到本地 `docker/data/rabbitmq` 目录
- **Nginx**：运行在 `nginx:1.20` 镜像，负责反向代理和静态文件服务

### 2. 生产环境

#### 2.1 后端服务
- **构建方式**：多阶段构建，使用 `golang:1.24-alpine` 构建，`alpine:latest` 运行
- **运行方式**：使用预构建的可执行文件 `goshopadmin-server`
- **访问地址**：通过Nginx反向代理

#### 2.2 前端服务
- **构建方式**：使用 `node:20-alpine` 镜像执行 `npm run build`
- **构建产物**：生成的 `dist` 文件存储在 `frontend/dist` 目录
- **运行方式**：Nginx从 `frontend/dist` 目录提供静态文件服务
- **访问地址**：通过Nginx反向代理

#### 2.3 部署架构
- **Nginx**：作为反向代理，处理静态文件和API请求转发
- **容器编排**：使用Docker Compose管理所有服务
- **数据持久化**：所有服务数据都持久化到本地目录

## 三、开发流程

### 1. 环境准备

1. **安装Docker**：确保本地安装了Docker和Docker Compose
2. **克隆代码**：从GitHub克隆项目代码
3. **配置环境变量**：
   - 后端：复制 `.env.example` 为 `.env` 并修改配置
   - 前端：无需特殊配置

### 2. 启动开发环境

```bash
# 进入docker目录
cd docker

# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 3. 开发工作流

#### 3.1 后端开发
1. 修改 `backend` 目录下的代码
2. 代码会自动同步到容器中
3. Go服务会自动重新运行
4. 通过 `http://localhost:8080` 访问API

#### 3.2 前端开发
1. 修改 `frontend` 目录下的代码
2. 代码会自动同步到容器中
3. Vite会自动热更新
4. 通过 `http://localhost:3000` 访问前端

### 4. 代码提交

1. **代码规范**：遵循Go和Vue的代码规范
2. **提交信息**：使用清晰的提交信息
3. **分支管理**：遵循GitFlow分支管理规范

## 四、部署流程

### 1. 构建生产镜像

```bash
# 进入docker目录
cd docker

# 构建所有服务
docker-compose build
```

### 2. 部署生产环境

#### 2.1 配置修改
- 修改 `docker-compose.yml` 中的环境变量
- 确保数据持久化目录存在
- 配置Nginx反向代理

#### 2.2 启动服务

```bash
# 启动生产服务（设置环境变量）
NODE_ENV=production RESTART_POLICY=always docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 3. 部署验证

1. **后端API**：访问 `http://服务器IP:8080/health` 验证健康状态
2. **前端应用**：访问 `http://服务器IP` 验证前端是否正常
3. **数据库连接**：验证MySQL、Redis、RabbitMQ连接正常

### 4. 监控与维护

1. **日志管理**：使用 `docker-compose logs` 查看服务日志
2. **服务监控**：配置健康检查和监控系统
3. **备份策略**：定期备份 `docker/data` 目录
4. **更新部署**：
   - 停止服务：`docker-compose down`
   - 拉取最新代码：`git pull`
   - 重新构建：`docker-compose build`
   - 启动服务：`docker-compose up -d`

## 五、常见问题与解决方案

### 1. 服务启动失败
- **检查日志**：`docker-compose logs 服务名`
- **端口冲突**：确保端口未被占用
- **依赖问题**：检查数据库和中间件是否正常

### 2. 代码同步问题
- **检查挂载卷**：确保本地目录正确挂载
- **权限问题**：检查文件权限
- **路径问题**：确保路径配置正确

### 3. 性能优化
- **资源限制**：在 `docker-compose.yml` 中配置资源限制
- **缓存策略**：合理配置Redis缓存
- **数据库优化**：添加索引，优化查询

### 4. 安全配置
- **环境变量**：敏感信息使用环境变量
- **网络隔离**：使用Docker网络隔离服务
- **访问控制**：配置Nginx访问控制
