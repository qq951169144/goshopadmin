### 1.3 数据模型分析

#### 商品模型 (Product)
```go
type Product struct {
    ID          uint      // 商品ID
    Name        string    // 商品名称
    Description string    // 商品描述
    Price       float64   // 价格
    SKU         string    // SKU编码
    Stock       int       // 库存
    Image       string    // 主图
}
```

#### 当前API问题
**`GET /api/products` 接口** ([product_controller.go#L25-46](file:///d:/code/goshopadmin/shop-backend/controllers/product_controller.go#L25-L46))：
- 当前只查询 `products` 表
- **需要关联查询 `product_skus` 表**，获取默认SKU的价格
- **需要返回商品主图** `image` 字段
- 建议返回字段：
  - `id`: 商品ID
  - `name`: 商品名称
  - `image`: 商品主图
  - `default_sku_price`: 默认SKU价格（取第一个SKU的价格）
  - `sales`: 销量

## 三、技术方案

### 3.1 前端技术栈
- **框架**：Vue 3 + Composition API
- **UI库**：Element Plus
- **路由**：Vue Router 4
- **状态管理**：Pinia
- **HTTP**：Axios
- **构建工具**：Vite

### 3.2 后端技术栈
- **框架**：Go + Gin
- **ORM**：GORM
- **数据库**：MySQL
- **缓存**：Redis（可选）

### 3.3 接口规范

#### 商品列表接口
```
GET /api/products

请求参数：
- page: 页码，默认1
- limit: 每页数量，默认10
- keyword: 搜索关键词（可选）
- category_id: 分类ID（可选）

响应格式：
{
  "code": 0,
  "message": "success",
  "data": {
    "products": [
      {
        "id": 1,
        "name": "商品名称",
        "image": "主图URL",
        "default_sku_price": 99.99,  // 默认SKU价格
        "sales": 100  // 销量
      }
    ],
    "total": 100
  }
}
```

**后端实现说明** ([product_controller.go#L25-46](file:///d:/code/goshopadmin/shop-backend/controllers/product_controller.go#L25-L46))：
1. 查询 `products` 表获取商品基本信息
2. 关联查询 `product_skus` 表，按 `product_id` 分组取第一条SKU记录
3. 将SKU的 `price` 作为 `default_sku_price` 返回
4. 返回 `image` 字段作为商品主图

#### 商品详情接口
```
GET /api/products/:id

响应格式：
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "name": "商品名称",
    "description": "商品描述",
    "price": 99.99,
    "images": ["图1", "图2", "图3"],
    "detail": "<html>商品详情</html>",
    "skus": [
      {
        "id": 1,
        "name": "红色-M",
        "price": 99.99,
        "stock": 100,
        "attributes": {"颜色": "红色", "尺码": "M"}
      }
    ],
    "sales": 1000,
    "reviews_count": 50
  }
}
```

## 七、API接口补充

### 7.1 需要新增的接口

> **命名规范**：与用户相关的接口统一使用 `customer` 前缀，保持与现有 `/api/customer/profile`、`/api/customer/orders` 一致

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 获取地址列表 | GET | `/api/customer/addresses` | 获取用户所有地址 |
| 添加地址 | POST | `/api/customer/addresses` | 新增收货地址 |
| 更新地址 | PUT | `/api/customer/addresses/:id` | 修改收货地址 |
| 删除地址 | DELETE | `/api/customer/addresses/:id` | 删除收货地址 |
| 设置默认地址 | PUT | `/api/customer/addresses/:id/default` | 设置默认地址 |
| 获取分类列表 | GET | `/api/categories` | 获取商品分类 |
| 取消订单 | POST | `/api/orders/:id/cancel` | 取消订单 |
| 确认收货 | POST | `/api/orders/:id/confirm` | 确认收货 |
