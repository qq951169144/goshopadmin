# 商城后台多商户系统技术选型与搭建计划（最终版）

## 一、技术栈选型

### 1. 服务端技术栈
- **语言**: Go 1.20+
- **框架**: Gin (高性能、轻量级，适合REST API开发)
- **数据库**: MySQL 8.0+ (主数据库) + Redis 7.0+ (缓存、会话管理)
  - 后续规划: 实现MySQL读写分离与主从切换
  - 现阶段: 仅使用单一数据库实例
- **消息队列**: RabbitMQ 3.10+ (异步任务处理、订单状态同步)
- **Web服务器**: Nginx 1.20+ (反向代理、负载均衡)
- **容器化**: Docker + Docker Compose (环境一致性、快速部署)

### 2. 前端技术栈（已确认）
- **框架**: Vue 3 + Vite
- **UI组件库**: Element Plus
- **调试工具**: Vue DevTools、Vite调试模式
- **学习成本**: 中（官方文档完善，中文资源丰富）
- **优势**: 生态成熟，调试工具丰富，组件库完善，开发效率高

## 二、系统架构设计

### 1. 模块划分
- **认证模块**: 登录、权限管理、角色分配（含图片滑块验证）
- **商户模块**: 商户入驻、信息管理、资质审核
- **商品模块**: 商品发布、分类管理、库存管理
- **订单模块**: 订单处理、支付集成、物流跟踪
- **活动模块**: 秒杀活动、促销管理、优惠券系统
- **统计模块**: 销售数据、流量分析、报表生成

### 2. 数据库设计
- **用户表**: 存储管理员和商户账号信息
- **角色表**: 定义角色权限
- **商品表**: 商品基本信息
- **订单表**: 订单详情
- **活动表**: 活动配置信息
- **物流表**: 物流状态跟踪
