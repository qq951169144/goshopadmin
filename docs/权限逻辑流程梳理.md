### 2.2 模型定义

#### User 模型 (models/user.go)

* 每个用户属于一个角色 (RoleID 外键)

* 通过 `user.Role` 可访问角色信息

* 通过 `user.Role.Permissions` 可访问角色拥有的权限

#### Role 模型 (models/role.go)

* 角色与权限是多对多关系

* 中间表: `role_permissions`

* 通过 `role.Permissions` 可访问权限列表

* 通过 `role.Users` 可访问拥有该角色的用户

#### Permission 模型 (models/permission.go)

* 核心字段: `Code` (权限码，如 "user:manage")

* 权限码格式: `{模块}:{操作}` (如 user:manage, product:read)

* 通过 `permission.Roles` 可访问拥有该权限的角色

### 3.1 流程图

```
┌─────────────┐
│   用户请求   │
└──────┬──────┘
       │
       ▼
┌─────────────────────────────────────────────────────────────┐
│                    AuthMiddleware 认证中间件                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 获取 Authorization Header                              │
│     └─▶ 为空? ──YES──▶ 返回 401 "未提供认证信息"              │
│                                                             │
│  2. 解析 Bearer Token                                      │
│     └─▶ 格式错误? ──YES──▶ 返回 401 "认证格式错误"            │
│                                                             │
│  3. 解析 JWT Token (utils.ParseToken)                      │
│     └─▶ 无效? ──YES──▶ 返回 401 "无效的token"                │

│  4. 将用户信息存入 Gin Context                              │

│     └─▶ c.Set("roleID", claims.RoleID)                     │
│                                                             │
│  5. 调用 c.Next() 继续执行后续中间件/处理器                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 四、授权流程 (Authorization)

### 4.1 流程图

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                  PermissionMiddleware 权限验证中间件                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  输入: requiredPermissions []string (需要的权限码列表)                        │
│                                                                             │
│  1. 从 Context 获取 userID                                                 │
│     └─▶ 不存在或无效? ──YES──▶ 返回 401 "未认证"                             │
│                                                                             │
│  2. 获取用户权限列表 (getCachedPermissions)                                  │
│     ├─▶ 先查 Redis 缓存 (key: user:{userID}:permissions)                    │
│     │   └─▶ 命中? ──YES──▶ 直接返回缓存数据                                  │
│     │
│     └─▶ 缓存未命中                                                         │
│         ├─▶ 查询数据库 (permissionService.GetUserPermissionCodes)            │
│         │   └─▶ 通过 user.Role.Permissions 获取权限列表                      │
│         │
│         └─▶ 写入 Redis 缓存 (TTL: 1小时)                                    │
│                                                                             │
│  3. 权限验证 (hasPermission)                                                │
│     ├─▶ 失败? ──YES──▶ 返回 403 "权限不足"                                   │
│     └─▶ 成功 ──▶ 调用 c.Next() 继续执行                                      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4.2 权限验证逻辑详解

#### hasPermission 函数 (第168-206行)

支持两种权限验证模式:

**1. AND 逻辑 (默认)**

* 所有必需的权限都必须满足

* 示例: `PermissionMiddleware("user:read", "user:write")`

* 用户必须同时拥有 `user:read` 和 `user:write` 权限

**2. OR 逻辑 (使用** **`|`** **分隔)**

* 只要满足其中一个权限即可

* 示例: `PermissionMiddleware("user:read|user:admin")`

* 用户拥有 `user:read` 或 `user:admin` 任一权限即可

**3. 组合使用**

* 示例: `PermissionMiddleware("user:manage", "admin:super|system:root")`

* 用户必须拥有 `user:manage` 权限

* 且同时拥有 `admin:super` 或 `system:root` 之一

## 五、路由权限配置

### 5.1 路由分组结构 (routes/routes.go)

```
/api
├── /auth                    # 认证相关 (部分需要认证)
│   ├── POST /login          # 无需认证
│   ├── GET /captcha         # 无需认证
│   ├── POST /captcha/verify # 无需认证
│   ├── POST /logout         # 需要认证 (AuthMiddleware)
│   ├── POST /refresh        # 需要认证
│   └── GET /me              # 需要认证
│
└── /                        # 业务路由 (需要认证 + 权限)
    ├── /users               # user:manage
    ├── /roles               # role:manage
    ├── /permissions         # role:manage
    ├── /merchants           # merchant:manage
    ├── /products            # product:manage
    ├── /product-categories  # product:manage
    ├── /product-images      # product:manage
    ├── /specifications      # product:manage
    ├── /specification-values # product:manage
    └── /skus                # product:manage
```

### 5.2 权限码定义

| 权限码               | 说明      | 应用路由                                                                                                                   |
| ----------------- | ------- | ---------------------------------------------------------------------------------------------------------------------- |
| `user:manage`     | 用户管理权限  | /api/users                                                                                                             |
| `role:manage`     | 角色和权限管理 | /api/roles, /api/permissions                                                                                           |
| `merchant:manage` | 商户管理权限  | /api/merchants                                                                                                         |
| `product:manage`  | 商品管理权限  | /api/products, /api/product-categories, /api/product-images, /api/specifications, /api/specification-values, /api/skus |

## 六、权限服务 (PermissionService)

### 6.1 核心方法 (services/permission\_service.go)

| 方法                                         | 功能       | 说明                          |
| ------------------------------------------ | -------- | --------------------------- |
| `GetUserPermissions(userID)`               | 获取用户权限列表 | 通过 user.Role.Permissions 查询 |
| `GetUserPermissionCodes(userID)`           | 获取权限码数组  | 提取 Permission.Code          |
| `HasPermission(userID, code)`              | 检查单个权限   | 用于细粒度控制                     |
| `AssignPermissions(roleID, permissionIDs)` | 分配权限     | 更新角色权限                      |
| `ClearUserPermissionCache(userID)`         | 清除权限缓存   | 权限变更时调用                     |

### 6.2 权限变更时缓存处理

当角色权限发生变化时，需要清除相关用户的权限缓存:

```go
// 清除单个用户缓存
middleware.ClearPermissionCache(userID)

// 或调用服务方法
permissionService.ClearUserPermissionCache(userID)
```

## 九、安全注意事项

1. **JWT密钥管理**: 密钥通过 `SetJWTSecret()` 设置，必须在应用启动时配置
2. **缓存安全**: 权限缓存1小时，敏感操作后应及时清除缓存
3. **权限最小化**: 每个路由应只配置必要的最小权限
4. **Token过期**: JWT有过期时间，需要定期刷新
