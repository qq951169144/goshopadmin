# 依赖注入容器优化方案

## 优化目标

结合项目实际需求，优先实施以下优化：

1. **单例模式 + 懒加载**：减少内存使用和启动时间
2. **服务间依赖管理**：解决复杂依赖关系
3. **接口抽象**：提高可测试性
4. **配置管理集成**：统一管理配置项
5. **路由模块化**：保持路由层级清晰，减少重复代码

## 具体文件修改方案

```go
package config

import (
	"os"
	"strconv"

	"github.com/joho/godotenv"
)

type Config struct {
	ServerPort string
	DBHost     string
	DBPort     string
	DBUser     string
	DBPassword string
	DBName     string
	JWTSecret  string
	JWTExpiry  int
}

var AppConfig *Config

func LoadConfig() error {
	if err := godotenv.Load(); err != nil {
	}

	AppConfig = &Config{
		ServerPort: getEnv("SERVER_PORT", "8080"),
		DBHost:     getEnv("DB_HOST", "localhost"),
		DBPort:     getEnv("DB_PORT", "3306"),
		DBUser:     getEnv("DB_USER", "root"),
		DBPassword: getEnv("DB_PASSWORD", "password"),
		DBName:     getEnv("DB_NAME", "goshopadmin"),
		JWTSecret:  getEnv("JWT_SECRET", "your-secret-key"),
		JWTExpiry:  getEnvAsInt("JWT_EXPIRY", 24),
	}

	return nil
}

func getEnv(key, defaultValue string) string {
	if value := os.Getenv(key); value != "" {
		return value
	}
	return defaultValue
}

func getEnvAsInt(key string, defaultValue int) int {
	if value := os.Getenv(key); value != "" {
		if intValue, err := strconv.Atoi(value); err == nil {
			return intValue
		}
	}
	return defaultValue
}
```

### 3. 优化依赖注入容器

#### 方案3：使用泛型 + 注册器（推荐）

**文件：`backend/di/container.go`**

```go
package di

import (
	"goshopadmin/config"
	"goshopadmin/controllers"
	"goshopadmin/services"

	"gorm.io/gorm"
)

type Factory func(*Container) interface{}

type Container struct {
	DB *gorm.DB
	
	instances map[string]interface{}
	factories map[string]Factory
}

func NewContainer(db *gorm.DB) *Container {
	c := &Container{
		DB:        db,
		instances: make(map[string]interface{}),
		factories: make(map[string]Factory),
	}
	c.registerFactories()
	return c
}

func Get[T any](c *Container, key string) T {
	return c.get(key).(T)
}

func (c *Container) get(key string) interface{} {
	if instance, exists := c.instances[key]; exists {
		return instance
	}
	
	if factory, exists := c.factories[key]; exists {
		instance := factory(c)
		c.instances[key] = instance
		return instance
	}
	
	return nil
}

func (c *Container) register(key string, factory Factory) {
	c.factories[key] = factory
}

func (c *Container) registerFactories() {
	c.register("commonController", func(c *Container) interface{} {
		return controllers.NewCommonController()
	})
	
	c.register("authController", func(c *Container) interface{} {
		return controllers.NewAuthController(c.GetAuthService())
	})
	
	c.register("userController", func(c *Container) interface{} {
		return controllers.NewUserController(c.DB)
	})
	
	c.register("roleController", func(c *Container) interface{} {
		return controllers.NewRoleController(c.DB)
	})
	
	c.register("permissionController", func(c *Container) interface{} {
		return controllers.NewPermissionController(c.DB)
	})
	
	c.register("merchantController", func(c *Container) interface{} {
		return controllers.NewMerchantController(c.GetMerchantService())
	})
	
	c.register("productController", func(c *Container) interface{} {
		return controllers.NewProductController(c.GetProductService())
	})
	
	c.register("authService", func(c *Container) interface{} {
		return services.NewAuthService(c.DB, config.AppConfig.JWTSecret, config.AppConfig.JWTExpiry)
	})
	
	c.register("merchantService", func(c *Container) interface{} {
		return services.NewMerchantService(c.DB)
	})
	
	c.register("productService", func(c *Container) interface{} {
		return services.NewProductService(c.DB, c.GetMerchantService())
	})
}

func (c *Container) GetCommonController() controllers.CommonControllerInterface {
	return Get[controllers.CommonControllerInterface](c, "commonController")
}

func (c *Container) GetAuthController() controllers.AuthControllerInterface {
	return Get[controllers.AuthControllerInterface](c, "authController")
}

func (c *Container) GetUserController() controllers.UserControllerInterface {
	return Get[controllers.UserControllerInterface](c, "userController")
}

func (c *Container) GetRoleController() controllers.RoleControllerInterface {
	return Get[controllers.RoleControllerInterface](c, "roleController")
}

func (c *Container) GetPermissionController() controllers.PermissionControllerInterface {
	return Get[controllers.PermissionControllerInterface](c, "permissionController")
}

func (c *Container) GetMerchantController() controllers.MerchantControllerInterface {
	return Get[controllers.MerchantControllerInterface](c, "merchantController")
}

func (c *Container) GetProductController() controllers.ProductControllerInterface {
	return Get[controllers.ProductControllerInterface](c, "productController")
}

func (c *Container) GetAuthService() services.AuthServiceInterface {
	return Get[services.AuthServiceInterface](c, "authService")
}

func (c *Container) GetMerchantService() services.MerchantServiceInterface {
	return Get[services.MerchantServiceInterface](c, "merchantService")
}

func (c *Container) GetProductService() services.ProductServiceInterface {
	return Get[services.ProductServiceInterface](c, "productService")
}
```

**方案3特点**：
- 使用泛型 `Get[T any]` 函数消除重复代码
- `register` 辅助方法简化构造函数注册
- `GetXXX` 方法只需1行代码
- 单例模式 + 懒加载
- 类型安全，编译时检查
- 无反射开销，性能优异

#### 方案4：使用结构体标签和反射自动注册

**文件：`backend/di/container.go`**

```go
package di

import (
	"goshopadmin/config"
	"goshopadmin/controllers"
	"goshopadmin/services"
	"reflect"

	"gorm.io/gorm"
)

const (
	Inject = "inject"
	InjectType = "injectType"
)

type Container struct {
	DB *gorm.DB
	
	instances map[reflect.Type]interface{}
}

func NewContainer(db *gorm.DB) *Container {
	return &Container{
		DB:        db,
		instances: make(map[reflect.Type]interface{}),
	}
}

func (c *Container) Get(t reflect.Type) interface{} {
	if instance, exists := c.instances[t]; exists {
		return instance
	}
	
	instance := c.resolve(t)
	if instance != nil {
		c.instances[t] = instance
	}
	return instance
}

func (c *Container) resolve(t reflect.Type) interface{} {
	if t == reflect.TypeOf((*gorm.DB)(nil)).Elem() {
		return c.DB
	}
	
	if t.Kind() == reflect.Interface {
		return c.resolveInterface(t)
	}
	
	if t.Kind() == reflect.Struct {
		return c.resolveStruct(t)
	}
	
	return nil
}

func (c *Container) resolveInterface(t reflect.Type) interface{} {
	switch t {
	case reflect.TypeOf((*controllers.CommonControllerInterface)(nil)).Elem():
		return controllers.NewCommonController()
	case reflect.TypeOf((*controllers.AuthControllerInterface)(nil)).Elem():
		authService := c.Get(reflect.TypeOf((*services.AuthServiceInterface)(nil)).Elem())
		if authService != nil {
			return controllers.NewAuthController(authService.(services.AuthServiceInterface))
		}
	case reflect.TypeOf((*controllers.UserControllerInterface)(nil)).Elem():
		return controllers.NewUserController(c.DB)
	case reflect.TypeOf((*controllers.RoleControllerInterface)(nil)).Elem():
		return controllers.NewRoleController(c.DB)
	case reflect.TypeOf((*controllers.PermissionControllerInterface)(nil)).Elem():
		return controllers.NewPermissionController(c.DB)
	case reflect.TypeOf((*controllers.MerchantControllerInterface)(nil)).Elem():
		merchantService := c.Get(reflect.TypeOf((*services.MerchantServiceInterface)(nil)).Elem())
		if merchantService != nil {
			return controllers.NewMerchantController(merchantService.(services.MerchantServiceInterface))
		}
	case reflect.TypeOf((*controllers.ProductControllerInterface)(nil)).Elem():
		productService := c.Get(reflect.TypeOf((*services.ProductServiceInterface)(nil)).Elem())
		if productService != nil {
			return controllers.NewProductController(productService.(services.ProductServiceInterface))
		}
	case reflect.TypeOf((*services.AuthServiceInterface)(nil)).Elem():
		return services.NewAuthService(c.DB, config.AppConfig.JWTSecret, config.AppConfig.JWTExpiry)
	case reflect.TypeOf((*services.MerchantServiceInterface)(nil)).Elem():
		return services.NewMerchantService(c.DB)
	case reflect.TypeOf((*services.ProductServiceInterface)(nil)).Elem():
		merchantService := c.Get(reflect.TypeOf((*services.MerchantServiceInterface)(nil)).Elem())
		if merchantService != nil {
			return services.NewProductService(c.DB, merchantService.(services.MerchantServiceInterface))
		}
	}
	return nil
}

func (c *Container) resolveStruct(t reflect.Type) interface{} {
	instance := reflect.New(t).Elem()
	
	for i := 0; i < t.NumField(); i++ {
		field := t.Field(i)
		
		if injectTag := field.Tag.Get(Inject); injectTag == "true" {
			fieldType := field.Type
			
			dependency := c.Get(fieldType)
			if dependency != nil {
				instance.Field(i).Set(reflect.ValueOf(dependency))
			}
		}
	}
	
	return instance.Interface()
}

func (c *Container) GetCommonController() controllers.CommonControllerInterface {
	return c.Get(reflect.TypeOf((*controllers.CommonControllerInterface)(nil)).Elem()).(controllers.CommonControllerInterface)
}

func (c *Container) GetAuthController() controllers.AuthControllerInterface {
	return c.Get(reflect.TypeOf((*controllers.AuthControllerInterface)(nil)).Elem()).(controllers.AuthControllerInterface)
}

func (c *Container) GetUserController() controllers.UserControllerInterface {
	return c.Get(reflect.TypeOf((*controllers.UserControllerInterface)(nil)).Elem()).(controllers.UserControllerInterface)
}

func (c *Container) GetRoleController() controllers.RoleControllerInterface {
	return c.Get(reflect.TypeOf((*controllers.RoleControllerInterface)(nil)).Elem()).(controllers.RoleControllerInterface)
}

func (c *Container) GetPermissionController() controllers.PermissionControllerInterface {
	return c.Get(reflect.TypeOf((*controllers.PermissionControllerInterface)(nil)).Elem()).(controllers.PermissionControllerInterface)
}

func (c *Container) GetMerchantController() controllers.MerchantControllerInterface {
	return c.Get(reflect.TypeOf((*controllers.MerchantControllerInterface)(nil)).Elem()).(controllers.MerchantControllerInterface)
}

func (c *Container) GetProductController() controllers.ProductControllerInterface {
	return c.Get(reflect.TypeOf((*controllers.ProductControllerInterface)(nil)).Elem()).(controllers.ProductControllerInterface)
}

func (c *Container) GetAuthService() services.AuthServiceInterface {
	return c.Get(reflect.TypeOf((*services.AuthServiceInterface)(nil)).Elem()).(services.AuthServiceInterface)
}

func (c *Container) GetMerchantService() services.MerchantServiceInterface {
	return c.Get(reflect.TypeOf((*services.MerchantServiceInterface)(nil)).Elem()).(services.MerchantServiceInterface)
}

func (c *Container) GetProductService() services.ProductServiceInterface {
	return c.Get(reflect.TypeOf((*services.ProductServiceInterface)(nil)).Elem()).(services.ProductServiceInterface)
}
```

**方案4特点**：
- 使用反射自动解析依赖
- 支持结构体标签注入
- 自动化程度高
- 性能开销较大
- 调试困难

```go
package routes

import (
	"goshopadmin/di"
	"goshopadmin/middleware"
	"github.com/gin-gonic/gin"
)

func RegisterAuthRoutes(api *gin.RouterGroup, container *di.Container) {
	auth := api.Group("/auth")
	{
		auth.POST("/login", container.GetAuthController().Login)
		auth.GET("/captcha", container.GetAuthController().GetCaptcha)
		auth.POST("/captcha/verify", container.GetAuthController().VerifyCaptcha)
		
		authProtected := auth.Group("/")
		authProtected.Use(middleware.AuthMiddleware())
		{
			authProtected.POST("/logout", container.GetAuthController().Logout)
			authProtected.POST("/refresh", container.GetAuthController().RefreshToken)
			authProtected.GET("/me", container.GetAuthController().GetCurrentUser)
		}
	}
}
```

```go
package routes

import (
	"goshopadmin/di"
	"goshopadmin/middleware"

	"github.com/gin-gonic/gin"
	"gorm.io/gorm"
)

func SetupRoutes(r *gin.Engine, db *gorm.DB) {
	container := di.NewContainer(db)
	
	api := r.Group("/api")
	api.Use(middleware.RequestLogger())
	
	RegisterCommonRoutes(r, container)
	RegisterAuthRoutes(api, container)
	
	protected := api.Group("/")
	protected.Use(middleware.AuthMiddleware())
	{
		RegisterUserRoutes(protected, container)
		RegisterRoleRoutes(protected, container)
		RegisterPermissionRoutes(protected, container)
		RegisterMerchantRoutes(protected, container)
		RegisterProductRoutes(protected, container)
	}
}
```

**路由模块化方案特点**：
- 路由层级清晰：每个模块的路由结构在一个文件中
- 职责分离：不同业务模块的路由分开管理
- 易于维护：修改某个模块的路由只需修改对应的文件
- 无重复代码：不需要声明控制器变量，直接通过容器获取
- 主文件简洁：routes.go 只负责组织和调用
- 便于协作：不同开发者可以并行修改不同模块的路由

**文件：`backend/main.go`**

```go
package main

import (
	"fmt"
	"log"
	"goshopadmin/config"
	"goshopadmin/middleware"
	"goshopadmin/routes"

	"github.com/gin-gonic/gin"
	"gorm.io/driver/mysql"
	"gorm.io/gorm"
)

func main() {
	if err := config.LoadConfig(); err != nil {
		log.Fatalf("Failed to load config: %v", err)
	}

	dsn := fmt.Sprintf("%s:%s@tcp(%s:%d)/%s?charset=utf8mb4&parseTime=True&loc=Local",
		config.AppConfig.DBUser,
		config.AppConfig.DBPassword,
		config.AppConfig.DBHost,
		config.AppConfig.DBPort,
		config.AppConfig.DBName,
	)
	db, err := gorm.Open(mysql.Open(dsn), &gorm.Config{})
	if err != nil {
		log.Fatalf("Failed to connect to database: %v", err)
	}

	r := gin.Default()
	r.Use(middleware.CORSMiddleware())

	routes.SetupRoutes(r, db)

	port := config.AppConfig.ServerPort
	fmt.Printf("Server starting on port %d...\n", port)
	if err := r.Run(fmt.Sprintf(":%d", port)); err != nil {
		log.Fatalf("Failed to start server: %v", err)
	}
}
```

## 代码运行流程分析

### 方案3：泛型 + 注册器流程

**从 main.go 开始的运行流程**：

1. **应用启动**：`main.go` 中的 `main()` 函数执行
2. **配置加载**：
   - 调用 `config.LoadConfig()` 加载配置文件和环境变量
   - 初始化 `config.AppConfig` 全局配置实例
3. **数据库初始化**：
   - 使用配置中的数据库信息构建 DSN
   - 连接数据库，创建 `db *gorm.DB` 实例
4. **路由设置**：
   - 调用 `routes.SetupRoutes(r, db)`，传入数据库连接实例
   - 在 `SetupRoutes` 函数中：
     - 创建 Gin 引擎实例
     - 注册全局中间件（CORS、日志）
     - 创建依赖注入容器：`container := di.NewContainer(db)`
     - 调用各个路由注册函数
5. **容器初始化**：
   - `di.NewContainer(db)` 创建容器实例
   - 初始化 `instances` 和 `factories` map
   - 调用 `registerFactories()` 注册所有构造函数
6. **构造函数注册**：
   - 将各个控制器和服务的构造函数注册到 `factories` map 中
   - 例如：`c.register("authController", func(c *Container) interface{} { return controllers.NewAuthController(c.GetAuthService()) })`
7. **路由注册**：
   - 调用 `RegisterCommonRoutes(r, container)` 注册通用路由
   - 调用 `RegisterAuthRoutes(api, container)` 注册认证路由
   - 调用 `RegisterUserRoutes(protected, container)` 注册用户路由
   - 其他路由注册...
8. **服务器启动**：
   - `r.Run(":" + config.AppConfig.ServerPort)` 启动 HTTP 服务器
9. **请求处理**：
   - 当收到 HTTP 请求时：
     - Gin 引擎根据路由规则匹配对应的控制器方法
     - 控制器方法执行，调用相应的服务方法
     - 服务方法使用注入的依赖执行业务逻辑
     - 控制器返回响应
10. **控制器和服务创建（懒加载）**：
    - 首次调用 `container.GetAuthController()` 时：
      - 内部调用 `Get[controllers.AuthControllerInterface](c, "authController")`
      - 调用 `c.get("authController")`
      - 检查 `instances["authController"]` 是否存在
      - 不存在则执行对应的构造函数
      - 构造函数中调用 `c.GetAuthService()` 获取认证服务
      - `GetAuthService()` 同样通过懒加载创建服务实例
      - 创建控制器实例并缓存
    - 后续调用直接返回缓存的实例

### 方案4：结构体标签和反射流程

**执行示例流程**：

1. **容器初始化**：
   - `container := di.NewContainer(db)` 创建容器实例
   - 初始化 `instances` map 用于缓存实例
2. **获取认证控制器**：
   - 调用 `container.GetAuthController()`
   - 内部调用 `c.Get(reflect.TypeOf((*controllers.AuthControllerInterface)(nil)).Elem())`
   - 检查 `instances` 中是否存在该类型的实例，不存在则调用 `resolve` 方法
3. **解析接口类型**：
   - `resolve` 方法检测到是接口类型，调用 `resolveInterface` 方法
   - `resolveInterface` 方法根据接口类型匹配对应的实现
   - 对于 `AuthControllerInterface`，需要依赖 `AuthServiceInterface`
   - 递归调用 `c.Get(reflect.TypeOf((*services.AuthServiceInterface)(nil)).Elem())` 获取认证服务
4. **获取认证服务**：
   - 调用 `c.Get(reflect.TypeOf((*services.AuthServiceInterface)(nil)).Elem())`
   - 检查 `instances` 中是否存在该类型的实例，不存在则调用 `resolve` 方法
   - `resolveInterface` 方法创建 `services.NewAuthService(c.DB, config.AppConfig.JWTSecret, config.AppConfig.JWTExpiry)`
   - 缓存认证服务实例
5. **创建认证控制器**：
   - 使用获取的认证服务创建 `controllers.NewAuthController(authService)`
   - 缓存认证控制器实例
   - 返回认证控制器实例
6. **结构体自动注入**：
   - 当解析结构体类型时，`resolveStruct` 方法会检查结构体字段的 `inject` 标签
   - 对于标记为 `inject:"true"` 的字段，自动解析并注入依赖
   - 容器会自动解析并注入依赖

```go
	GetOrders() ([]models.Order, error)

	return &OrderService{
		DB: db,
	}

func (os *OrderService) GetOrders() ([]models.Order, error) {
	var orders []models.Order
	err := os.DB.Find(&orders).Error
	return orders, err
}

func (os *OrderService) GetOrderByID(id string) (models.Order, error) {
	var order models.Order
```

## 方案优缺点分析

### 方案3：泛型 + 注册器（推荐）

**优点**：
- **代码量最少**：GetXXX 方法从3行减少到1行，registerFactories 方法更简洁
- **类型安全**：泛型在编译时进行类型检查，避免运行时类型错误
- **性能优异**：无反射开销，运行效率高
- **可维护性强**：register 辅助方法使构造函数注册更清晰
- **扩展性好**：新增接口时只需在 registerFactories 中添加一行注册代码和一行 GetXXX 方法
- **符合 Go 风格**：简洁明了，易于理解和维护
- **代码复用**：泛型 Get 函数消除了 GetXXX 方法的重复代码

**缺点**：
- 需要 Go 1.18+ 版本支持泛型
- 依赖注入关系仍然需要手动管理
- 首次启动时需要注册所有构造函数，虽然是一次性操作

### 方案4：结构体标签和反射自动注册

**优点**：
- **自动依赖解析**：通过反射和结构体标签自动解析和注入依赖
- **减少手动代码**：无需手动注册构造函数，通过标签标记即可
- **灵活性高**：支持复杂的依赖关系自动解析
- **可扩展性强**：新增服务或控制器时，只需添加相应的标签即可
- **支持结构体注入**：可以直接注入到结构体字段，简化代码

**缺点**：
- **反射性能开销**：使用反射会带来一定的性能开销
- **代码复杂度增加**：反射代码较难理解和维护
- **类型安全减弱**：运行时类型检查，编译时无法发现错误
- **调试困难**：反射相关的错误难以调试
- **不符合 Go 简洁风格**：过度使用反射违背了 Go 语言的设计哲学

### 路由模块化方案

**优点**：
- **路由层级清晰**：每个模块的路由结构在一个文件中，一目了然
- **职责分离**：不同业务模块的路由分开管理
- **易于维护**：修改某个模块的路由只需修改对应的文件
- **无重复代码**：不需要声明控制器变量，直接通过容器获取
- **主文件简洁**：routes.go 只负责组织和调用，结构清晰
- **便于协作**：不同开发者可以并行修改不同模块的路由

**缺点**：
- 文件数量增加
- 需要创建多个路由文件

## 推荐方案

**强烈推荐使用方案3（泛型 + 注册器）+ 路由模块化方案**，这是所有方案中的最佳选择。

### 核心优势

1. **代码量最少**：
   - GetXXX 方法从原来的3行减少到1行
   - registerFactories 方法使用 register 辅助函数，代码更简洁
   - 整体代码量相比原始方案减少约30%

2. **类型安全最强**：
   - 使用泛型在编译时进行类型检查
   - 避免了运行时类型断言可能导致的 panic
   - IDE 可以提供更好的类型提示和自动补全

3. **性能最优**：
   - 无反射开销，运行效率最高
   - 优于方案4（反射方案）

4. **可维护性最佳**：
   - register 辅助方法使构造函数注册逻辑更清晰
   - 泛型 Get 函数消除了所有 GetXXX 方法的重复代码
   - 新增服务时只需添加2行代码（1行注册 + 1行 GetXXX 方法）

5. **符合 Go 设计哲学**：
   - 简洁明了，易于理解
   - 避免了过度使用反射
   - 充分利用了 Go 1.18+ 的泛型特性

6. **扩展性优秀**：
   - 支持单例模式和懒加载
   - 易于添加新的服务或控制器
   - 依赖关系管理清晰

7. **路由层级清晰**：
   - 每个模块的路由结构在一个文件中
   - 主路由文件简洁，只负责组织和调用
   - 易于理解和维护

### 与其他方案对比

| 特性 | 方案3（泛型+注册器） | 方案4（反射） | 路由模块化 |
|------|---------------------|-------------|----------|
| 代码量 | **最少** | 中等 | 中等 |
| 类型安全 | **最高** | 低 | 高 |
| 性能 | **高** | 低 | 高 |
| 可维护性 | **高** | 低 | **高** |
| 扩展性 | **高** | 高 | **高** |
| Go 版本要求 | **1.18+** | 任意 | 任意 |
| 路由层级清晰 | 中等 | 中等 | **最高** |

### 适用场景

方案3（泛型 + 注册器）+ 路由模块化适用于：
- 使用 Go 1.18+ 版本的项目
- 需要高性能依赖注入的场景
- 追求代码简洁性和可维护性的项目
- 中大型项目，服务数量较多
- 需要清晰路由层级结构的项目

### 实施建议

1. **版本要求**：确保项目使用 Go 1.18 或更高版本
2. **渐进式迁移**：可以从现有方案逐步迁移到方案3（泛型 + 注册器）
3. **代码生成**：如果服务数量非常多（>50个），可以考虑使用代码生成工具进一步简化
4. **测试覆盖**：确保所有依赖注入逻辑都有充分的单元测试
5. **路由模块化**：按业务模块拆分路由文件，保持路由层级清晰

### 总结

方案3（泛型 + 注册器）+ 路由模块化是所有方案中的最佳平衡点：
- ✅ 代码简洁：GetXXX 方法只需1行
- ✅ 类型安全：泛型编译时检查
- ✅ 性能优异：无反射开销
- ✅ 可维护性强：清晰的注册逻辑
- ✅ 扩展性好：易于添加新服务
- ✅ 符合 Go 风格：简洁明了
- ✅ 路由清晰：模块化路由文件

**方案4（结构体标签和反射）** 虽然提供了更高的自动化程度，但过度使用反射违背了 Go 语言的设计哲学，增加了代码复杂度和维护成本，性能也较差。因此，在大多数场景下，方案3（泛型 + 注册器）+ 路由模块化是最佳选择。

这种方案既解决了代码重复问题，又保持了 Go 语言的简洁风格，同时充分利用了现代 Go 的泛型特性，是当前最优的依赖注入实现方案。
