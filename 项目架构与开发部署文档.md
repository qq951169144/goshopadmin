# 商城后台多商户系统项目架构与开发部署文档

## 1. 项目概述

商城后台多商户系统是一个基于前后端分离架构的电子商务管理系统，支持多商户管理、商品管理、订单管理等功能。

## 2. 技术架构

### 2.1 整体架构

- **架构模式**：前后端分离架构
- **部署方式**：Docker容器化部署
- **网络架构**：微服务架构（基于Docker网络）

| | GORM | 1.25.7 | ORM框架 |
| | MySQL | 8.0 | 关系型数据库 |
| | JWT | 3.2.0 | 身份认证 |

## 4. 核心架构设计

### 4.1 后端架构

- **分层架构**：
  - 路由层（routes）：处理HTTP请求路由
  - 控制器层（controllers）：处理请求逻辑
  - 服务层（services）：业务逻辑处理
  - 模型层（models）：数据模型定义
  - 工具层（utils）：通用工具函数
  - 中间件层（middleware）：请求拦截和处理

- **核心流程**：
  1. 客户端发送HTTP请求
  2. 路由层接收请求并转发到对应控制器
  3. 控制器调用服务层处理业务逻辑
  4. 服务层操作数据库或调用其他服务
  5. 控制器返回响应给客户端

### 4.2 前端架构

- **组件化设计**：
  - 根组件（App.vue）：应用入口
  - 页面组件（Home.vue, Login.vue）：页面级组件
  - 通用组件（HelloWorld.vue）：可复用组件

- **API调用**：
  - 集中管理API调用（api目录）
  - 使用Axios进行HTTP请求

### 4.3 数据架构

- **数据库表结构**：
  - users：用户表
  - roles：角色表
  - permissions：权限表
  - role_permissions：角色权限关联表
  - products：商品表
  - orders：订单表
  - activities：活动表
  - shipping：物流表

- **数据流动**：
  - 前端 → API请求 → 后端控制器 → 服务层 → 数据库
  - 数据库 → 服务层 → 控制器 → API响应 → 前端

## 5. 开发流程

### 5.1 后端开发流程

1. **环境准备**：
   - 安装Go 1.24.5
   - 安装MySQL 8.0
   - 安装Redis 7.0
   - 安装RabbitMQ 3.10

2. **项目初始化**：
   - 克隆代码仓库
   - 进入backend目录
   - 执行 `go mod download` 下载依赖

3. **开发步骤**：
   - 定义数据模型（models）
   - 实现服务层逻辑（services）
   - 编写控制器（controllers）
   - 配置路由（routes）
   - 添加中间件（middleware）
   - 编写工具函数（utils）

4. **本地运行**：
   - 执行 `go run main.go` 启动服务
   - 服务默认运行在 `http://localhost:8080`

### 5.2 前端开发流程

1. **环境准备**：
   - 安装Node.js 20+
   - 安装npm或yarn

2. **项目初始化**：
   - 进入frontend目录
   - 执行 `npm install` 安装依赖

3. **开发步骤**：
   - 创建组件（components）
   - 编写API调用（api）
   - 实现页面逻辑
   - 添加样式

4. **本地运行**：
   - 执行 `npm run dev` 启动开发服务器
   - 服务默认运行在 `http://localhost:3000`

5. **构建生产版本**：
   - 执行 `npm run build` 生成静态文件
   - 构建产物存储在 `frontend/dist` 目录

### 5.3 开发规范

- **代码风格**：
  - 后端：遵循Go语言规范
  - 前端：遵循Vue 3风格指南

- **命名规范**：
  - 后端：使用驼峰命名法
  - 前端：组件名使用大驼峰，变量和方法使用小驼峰

- **提交规范**：
  - 使用语义化提交信息
  - 提交前运行代码检查

## 6. 部署流程

### 6.1 本地部署（开发环境）

1. **环境准备**：
   - 安装Docker
   - 安装Docker Compose

2. **启动服务**：
   - 进入docker目录
   - 执行 `docker-compose up -d` 启动所有服务
   - 服务启动后可通过以下地址访问：
     - 前端：`http://localhost:3000`
     - 后端API：`http://localhost:8080/api`
     - MySQL：`localhost:3306`
     - Redis：`localhost:6379`
     - RabbitMQ管理界面：`http://localhost:15672`

3. **停止服务**：
   - 执行 `docker-compose down` 停止所有服务

### 6.2 生产部署

1. **构建镜像**：
   - 后端：使用Docker Compose自动构建Go应用
   - 前端：使用Docker Compose自动构建Vue应用静态文件

2. **配置调整**：
   - 设置环境变量（生产环境配置）
   - 调整Nginx配置
   - 设置SSL证书

3. **部署步骤**：
   - 进入docker目录
   - 运行 `NODE_ENV=production RESTART_POLICY=always docker-compose up -d` 启动服务
   - 配置域名和DNS
   - 启动监控和日志收集

4. **构建产物**：
   - 前端构建的静态文件存储在 `frontend/dist` 目录
   - Nginx直接从该目录提供静态文件服务

### 6.3 服务说明

| 服务 | 容器名 | 端口 | 说明 |
|------|--------|------|------|
| MySQL | goshopadmin-mysql | 3306 | 关系型数据库 |
| Redis | goshopadmin-redis | 6379 | 缓存服务 |
| RabbitMQ | goshopadmin-rabbitmq | 5672/15672 | 消息队列/管理界面 |
| 后端 | goshopadmin-backend | 8080 | Go后端服务 |
| 前端 | goshopadmin-frontend | 3000 | Vue前端开发服务（开发环境）/ 构建静态文件（生产环境） |
| Nginx | goshopadmin-nginx | 80/443 | 反向代理和静态文件服务 |
