# 商城后台多商户系统项目架构与开发部署文档

## 1. 项目概述

商城后台多商户系统是一个基于前后端分离架构的电子商务管理系统，支持多商户管理、商品管理、订单管理等功能。

## 2. 技术架构

### 2.1 整体架构

- **架构模式**：前后端分离架构
- **部署方式**：Docker容器化部署
- **网络架构**：微服务架构（基于Docker网络）

### 2.2 技术栈

| 分类 | 技术 | 版本 | 用途 |
|------|------|------|------|
| **后端** | Go | 1.24.5 | 服务端开发语言 |
| | Gin | 1.11.0 | Web框架 |
| | GORM | 1.25.7 | ORM框架 |
| | MySQL | 8.0 | 关系型数据库 |
| | JWT | 3.2.0 | 身份认证 |
| | Redis | 7.0 | 缓存服务 |
| | RabbitMQ | 3.10 | 消息队列 |
| **前端** | Vue | 3.5.25 | 前端框架 |
| | Element Plus | 2.13.2 | UI组件库 |
| | Axios | 1.13.6 | HTTP客户端 |
| | Vite | 7.3.1 | 构建工具 |
| **部署** | Docker | - | 容器化平台 |
| | Docker Compose | 3.8 | 多容器编排 |
| | Nginx | 1.20 | 反向代理 |

## 4. 核心架构设计

### 4.1 后端架构

- **分层架构**：
  - 路由层（routes）：处理HTTP请求路由
  - 控制器层（controllers）：处理请求逻辑
  - 服务层（services）：业务逻辑处理
  - 模型层（models）：数据模型定义
  - 工具层（utils）：通用工具函数
  - 中间件层（middleware）：请求拦截和处理

- **核心流程**：
  1. 客户端发送HTTP请求
  2. 路由层接收请求并转发到对应控制器
  3. 控制器调用服务层处理业务逻辑
  4. 服务层操作数据库或调用其他服务
  5. 控制器返回响应给客户端

### 4.2 前端架构

- **组件化设计**：
  - 根组件（App.vue）：应用入口
  - 页面组件（Home.vue, Login.vue）：页面级组件
  - 通用组件（HelloWorld.vue）：可复用组件

- **API调用**：
  - 集中管理API调用（api目录）
  - 使用Axios进行HTTP请求

### 4.3 数据架构

- **数据库表结构**：
  - users：用户表
  - roles：角色表
  - permissions：权限表
  - role_permissions：角色权限关联表
  - products：商品表
  - orders：订单表
  - activities：活动表
  - shipping：物流表

- **数据流动**：
  - 前端 → API请求 → 后端控制器 → 服务层 → 数据库
  - 数据库 → 服务层 → 控制器 → API响应 → 前端

## 5. 环境运行逻辑

### 5.1 开发环境

#### 5.1.1 后端服务
- **运行方式**：使用 `golang:1.24-alpine` 镜像
- **代码同步**：本地 `backend` 目录挂载到容器 `/app` 目录
- **运行命令**：`go run main.go`
- **热更新**：本地修改代码后，需要重启后端容器以加载新代码
- **访问地址**：`http://localhost:8080`

#### 5.1.2 前端服务（管理端）
- **运行方式**：使用 `node:20-alpine` 镜像
- **代码同步**：本地 `frontend` 目录挂载到容器 `/app` 目录
- **运行命令**：根据 `NODE_ENV` 环境变量自动选择
  - 开发环境：`npm run dev -- --host`
  - 生产环境：`npm run build`
- **热更新**：Vite开发服务器支持热更新，若热更新不生效，需要重启前端容器
- **访问地址**：`http://localhost:3000` (开发环境)
- **构建产物**：生产环境构建的 `dist` 文件存储在 `frontend/dist` 目录
- **API配置**：
  - 前端API基础路径：`/api`
  - 开发环境代理：通过Vite代理将 `/api` 请求转发到后端服务
  - 生产环境代理：通过Nginx反向代理将 `/api` 请求转发到后端服务

#### 5.1.3 C端商城服务
- **运行方式**：使用 `node:20-alpine` 镜像
- **代码同步**：本地 `c端` 目录挂载到容器 `/app` 目录

- **访问地址**：`http://localhost:3001` (开发环境)
- **构建产物**：生产环境构建的 `dist` 文件存储在 `c端/dist` 目录
- **API配置**：

#### 5.2.4 Nginx配置

**文件**：`docker/nginx/nginx.conf`

**配置**：
```nginx
# 管理端
server {
    listen 80;
    server_name admin.example.com;

    root /usr/share/nginx/admin;
    index index.html;

    # 静态文件处理
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://backend:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# C端商城
server {
    listen 80;
    server_name example.com;

    root /usr/share/nginx/c端;
    index index.html;

    # 静态文件处理
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://backend:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

#### 5.2.5 部署架构
- **Nginx**：作为反向代理，处理静态文件和API请求转发
- **容器编排**：使用Docker Compose管理所有服务
- **数据持久化**：所有服务数据都持久化到本地目录

## 6. 开发流程

### 6.1 环境准备

1. **安装Docker**：确保本地安装了Docker和Docker Compose
2. **克隆代码**：从GitHub克隆项目代码
3. **配置环境变量**：
   - 后端：复制 `.env.example` 为 `.env` 并修改配置
   - 前端：无需特殊配置

### 6.2 启动开发环境

```bash
# 进入docker目录
cd docker

# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 6.3 开发工作流

1. 修改 `backend` 目录下的代码
2. 代码会自动同步到容器中
3. Go服务会自动重新运行

#### 6.3.2 前端开发（管理端）
1. 修改 `frontend` 目录下的代码
2. 代码会自动同步到容器中
3. Vite会自动热更新

#### 6.3.3 C端开发
1. 修改 `c端` 目录下的代码
2. 代码会自动同步到容器中
3. Vite会自动热更新
4. 通过 `http://localhost:3001` 访问C端

### 6.4 代码提交

1. **代码规范**：遵循Go和Vue的代码规范
