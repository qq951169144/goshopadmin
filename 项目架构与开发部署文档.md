# 商城后台多商户系统项目架构与开发部署文档

## 1. 项目概述

商城后台多商户系统是一个基于前后端分离架构的电子商务管理系统，支持多商户管理、商品管理、订单管理等功能。

## 2. 技术架构

### 2.1 整体架构

- **架构模式**：前后端分离架构
- **部署方式**：Docker容器化部署
- **网络架构**：微服务架构（基于Docker网络）

## 4. 核心架构设计

### 4.1 后端架构

- **分层架构**：
  - 路由层（routes）：处理HTTP请求路由
  - 控制器层（controllers）：处理请求逻辑
  - 服务层（services）：业务逻辑处理
  - 模型层（models）：数据模型定义
  - 工具层（utils）：通用工具函数
  - 中间件层（middleware）：请求拦截和处理

- **核心流程**：
  1. 客户端发送HTTP请求
  2. 路由层接收请求并转发到对应控制器
  3. 控制器调用服务层处理业务逻辑
  4. 服务层操作数据库或调用其他服务
  5. 控制器返回响应给客户端

### 4.2 前端架构

- **组件化设计**：
  - 根组件（App.vue）：应用入口
  - 页面组件（Home.vue, Login.vue）：页面级组件
  - 通用组件（HelloWorld.vue）：可复用组件

- **API调用**：
  - 集中管理API调用（api目录）
  - 使用Axios进行HTTP请求

### 4.3 数据架构

- **数据库表结构**：
  - users：用户表
  - roles：角色表
  - permissions：权限表
  - role_permissions：角色权限关联表
  - products：商品表
  - orders：订单表
  - activities：活动表
  - shipping：物流表

- **数据流动**：
  - 前端 → API请求 → 后端控制器 → 服务层 → 数据库
  - 数据库 → 服务层 → 控制器 → API响应 → 前端

## 5. 环境运行逻辑

### 5.1 开发环境

| 服务 | 容器名 | 端口 | 说明 |
|------|--------|------|------|
| MySQL | goshopadmin-mysql | 3306 | 关系型数据库 |
| Redis | goshopadmin-redis | 6379 | 缓存服务 |
| RabbitMQ | goshopadmin-rabbitmq | 5672/15672 | 消息队列/管理界面 |
| 后端 | goshopadmin-backend | 8080 | Go后端服务 |
| 前端 | goshopadmin-frontend | 3000 | Vue前端开发服务 |
| C端 | goshopadmin-c端 | 3001 | C端商城开发服务 |
| Nginx | goshopadmin-nginx | 80/443 | 反向代理和静态文件服务 |

### 5.2 生产环境

- **后端服务**：多阶段构建，使用预构建的可执行文件运行
- **前端服务**：构建产物存储在 `dist` 目录，Nginx提供静态文件服务
- **C端服务**：构建产物存储在 `dist` 目录，Nginx提供静态文件服务
- **Nginx**：作为反向代理，处理静态文件和API请求转发

详细的开发和部署操作请参考 [开发与部署流程.md](./开发与部署流程.md)
