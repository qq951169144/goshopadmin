# 端口管理命令备忘录

## 1. 查看被系统排除的 TCP 端口范围

### 命令
```cmd
netsh interface ipv4 show excludedportrange protocol=tcp
```

### 作用
- 显示 Windows 系统中被保留或排除的 TCP 端口范围
- 这些端口可能被 Windows 服务、Hyper-V、WSL 等占用
- 当尝试绑定端口时出现 "端口被占用" 错误，可以通过此命令查看是否被系统排除

## 2. 重启 Windows NAT 服务

### 命令
```cmd
net stop winnat && net start winnat
```

### 作用
- 停止并重启 Windows NAT 服务
- 当遇到端口占用问题，特别是 Docker 容器无法绑定端口时，此命令可以释放被占用的端口
- 适用于解决 "端口被占用" 或 "无法绑定端口" 的错误

### 使用场景
1. Docker 容器启动时出现端口绑定错误
2. 应用程序无法启动，提示端口被占用
3. 系统重启后端口仍然被占用

### 操作步骤
1. 以管理员身份打开命令提示符或 PowerShell
2. 执行 `net stop winnat` 停止 NAT 服务
3. 执行 `net start winnat` 启动 NAT 服务
4. 重新尝试启动应用程序或容器

## 注意事项

1. **权限要求**：执行这些命令需要管理员权限
2. **服务影响**：重启 winnat 服务可能会暂时影响网络连接，特别是使用 WSL 或 Hyper-V 的用户
3. **临时解决方案**：重启 winnat 服务是临时解决方案，如果端口占用问题频繁出现，可能需要检查系统服务或应用程序配置
4. **端口范围**：如果需要使用特定端口，建议选择未被系统排除的端口范围

## 示例场景

### 场景：Docker 容器无法绑定 5672 端口

1. 执行 `netsh interface ipv4 show excludedportrange protocol=tcp` 查看是否 5672 端口在排除范围内
2. 如果在排除范围内，执行 `net stop winnat && net start winnat` 重启 NAT 服务
3. 再次查看端口排除范围，确认 5672 端口已释放
4. 重新启动 Docker 容器

### 场景：应用程序无法绑定 8080 端口

1. 执行 `netsh interface ipv4 show excludedportrange protocol=tcp` 查看 8080 端口状态
2. 执行 `net stop winnat && net start winnat` 释放端口
3. 重新启动应用程序
