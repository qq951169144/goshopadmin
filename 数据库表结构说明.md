# 数据库表结构说明

## 1. 数据库基本信息

- **数据库名称**: goshopadmin
- **MySQL 版本**: 8.0.45
- **字符集**: utf8mb4
- **排序规则**: utf8mb4_unicode_ci

## 2. 表结构详情

### 2.1 users 表

**描述**: 用户表，存储系统用户信息

| 字段名 | 数据类型 | 约束 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | `int(0)` | `NOT NULL AUTO_INCREMENT` | - | 用户ID |
| `username` | `varchar(191)` | `NOT NULL UNIQUE` | - | 用户名 |
| `password` | `longtext` | `NOT NULL` | - | 密码（加密存储） |
| `role_id` | `int(0)` | `NOT NULL` | - | 角色ID |
| `status` | `enum('active','inactive')` | `NULL` | `'active'` | 用户状态 |
| `created_at` | `datetime(3)` | `NULL` | - | 创建时间 |
| `updated_at` | `datetime(3)` | `NULL` | - | 更新时间 |
| `email` | `varchar(191)` | `UNIQUE` | - | 邮箱 |

**索引**: 
- 主键: `id`
- 唯一索引: `username`, `email`
- 普通索引: `idx_users_role_id` (`role_id`), `idx_users_status` (`status`)

**引擎**: InnoDB

## 5. 数据库设计说明

1. **用户权限系统**: 采用 RBAC (Role-Based Access Control) 模型，通过 `roles`、`permissions` 和 `role_permissions` 表实现权限管理。

2. **商户管理系统**: 包含商户信息管理、商户审核、银行信息管理、提现管理和对账管理等功能。

3. **商品管理系统**: 支持商品分类、商品SKU、商品图片等功能，实现多维度的商品信息管理。

4. **订单系统**: 包含订单管理、订单明细、支付记录、地址管理等功能，支持完整的订单流程。

5. **活动系统**: 支持活动规则配置、活动商品管理和活动效果统计，实现丰富的营销活动功能。

6. **支付与物流系统**: 支持多种支付方式和物流方式的配置。

7. **数据关系**:
   - 用户与角色是多对一关系
   - 角色与权限是多对多关系
   - 商户与用户是多对多关系（通过merchant_users表）
   - 商户与商品是一对多关系
   - 商户与活动是一对多关系
   - C端用户与订单是一对多关系
   - 商户与订单是一对多关系
   - 订单与订单明细是一对多关系
   - 订单与支付是一对一关系
   - 订单与物流是一对一关系
   - C端用户与地址是一对多关系

8. **索引设计**: 为外键字段和常用查询字段创建了索引，提高查询性能。

9. **数据完整性**: 由应用代码控制数据的完整性和一致性，不再依赖数据库外键约束。

10. **时间戳**: 所有表都包含 `created_at` 和 `updated_at` 字段，记录数据的创建和更新时间。

11. **状态管理**: 对于需要状态管理的表（如用户、产品、活动、商户等），使用 `enum` 类型定义状态选项。

## 6. 注意事项

1. **密码安全**: 用户密码使用 bcrypt 算法加密存储，确保安全性。

2. **外键约束**: 已移除所有外键约束，数据完整性由应用代码控制，提高开发灵活性。

3. **字符集**: 使用 `utf8mb4` 字符集，支持表情符号等特殊字符。

4. **数据类型**: 合理选择数据类型，如使用 `decimal` 存储金额，确保精度。

5. **索引优化**: 为常用查询字段创建索引，提高查询性能。

6. **JSON字段**: 对于复杂结构的数据，使用 JSON 类型存储，提高灵活性。

7. **商户隔离**: 所有商户相关的数据都包含 `merchant_id` 字段，确保数据隔离。

8. **C端用户**: 单独的 `customers` 表存储前端用户信息，与后台管理用户分离。

## 7. 设计说明

### 7.1 商户管理系统

1. **商户表 (`merchants`)**：
   - 存储商户基本信息
   - 包含审核状态和商户状态
   - 审核通过后商户状态自动变为active

2. **商户用户关联表 (`merchant_users`)**：
   - 关联商户和用户
   - 支持一个商户多个用户
   - 定义商户内角色(owner, manager, staff)

3. **商户审核表 (`merchant_audit`)**：
   - 记录商户注册和信息更新的审核流程
   - 使用JSON类型存储审核前后的数据
   - 支持审核状态跟踪

4. **商户银行信息表 (`merchant_bank`)**：
   - 存储商户银行账户信息
   - 支持设置默认账户
   - 一个商户可以有多个银行账户

5. **商户提现记录表 (`merchant_withdraw`)**：
   - 记录商户提现申请和处理状态
   - 包含提现金额、银行信息等
   - 支持提现状态跟踪

6. **商户对账单表 (`merchant_statement`)**：
   - 生成商户定期对账单
   - 记录时间段内的交易汇总
   - 支持对账状态跟踪

### 7.2 商品管理系统

1. **商品分类表 (`product_categories`)**：
   - 支持多级分类
   - 支持分类排序
   - 与商品表关联

2. **商品表 (`products`)**：
   - 存储商品基本信息
   - 包含商品名称、描述、详情等
   - 关联商品分类和商户
   - 支持富文本详情编辑

3. **商品规格表 (`product_specifications`)**：
   - 定义商品的规格维度（如颜色、尺寸、版本）
   - 支持规格排序，控制前端展示顺序
   - 与商品表一对多关联

4. **规格值表 (`product_specification_values`)**：
   - 存储规格的具体值（如红色、M码、标准版）
   - 支持规格值排序和状态管理
   - 与规格表一对多关联

5. **商品SKU表 (`product_skus`)**：
   - 存储商品的不同属性组合
   - 每个SKU有独立的编码、价格和库存
   - 支持原价和现价（用于促销）
   - 使用JSON存储SKU属性

6. **SKU规格关联表 (`product_sku_specs`)**：
   - 建立SKU与规格的关联关系
   - 记录SKU使用了哪些规格值
   - 支持多规格组合（如红色+M码）

7. **商品图片表 (`product_images`)**：
   - 存储商品的多张图片
   - 支持设置主图
   - 支持图片排序

#### 商品规格系统设计说明

**规格系统数据流：**

```
创建商品流程：
1. 创建商品基本信息 (products)
2. 创建商品规格定义 (product_specifications)
   - 如：颜色、尺寸
3. 创建规格值 (product_specification_values)
   - 颜色：红色、蓝色、黑色
   - 尺寸：S、M、L
4. 创建SKU (product_skus)
   - 红色-S、红色-M、红色-L
   - 蓝色-S、蓝色-M、蓝色-L
   - 黑色-S、黑色-M、黑色-L
5. 建立SKU与规格的关联 (product_sku_specs)
   - 每个SKU关联对应的规格值
```

| 商品 | 规格1(颜色) | 规格2(尺寸) | SKU编码 | 价格 | 库存 |

**前端展示逻辑：**
1. 获取商品详情时，同时获取规格列表和规格值列表
2. 用户选择规格值时，实时计算可选组合
3. 根据选择的规格组合，查找对应的SKU
4. 显示对应SKU的价格和库存

## 8. 业务逻辑与注意事项

### 8.1 数据一致性
- 商户审核通过后，自动更新商户状态
- 商户状态变更时，同步更新关联数据
- 订单状态变更时，同步更新支付和物流状态
- 商品库存变更时，同步更新SKU库存

### 8.2 性能考虑
- 为常用查询字段添加索引
- 合理设计表结构，避免冗余字段
- 对于JSON字段，考虑使用适当的索引策略
- 活动统计数据可考虑使用缓存机制

### 8.3 安全性
- 敏感信息如银行账号需要加密存储
- 提现操作需要多重验证
- 支付信息需要加密传输
- 商户审核流程需要严格的权限控制

### 8.4 扩展性
- 表结构设计预留扩展字段
- 权限系统支持后续功能扩展
- 商品属性和活动规则使用JSON格式，便于灵活配置
- 支付方式设计支持后续扩展

### 8.5 业务逻辑
- 活动商品库存管理需要独立于普通商品库存
- 订单创建时需要扣减相应的商品库存
- 订单取消时需要恢复商品库存
- 商户提现需要与对账单关联
