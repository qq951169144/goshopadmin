# 数据库表结构说明

## 1. 数据库基本信息

- **数据库名称**: goshopadmin
- **MySQL 版本**: 8.0.43
- **字符集**: utf8mb4
- **排序规则**: utf8mb4_unicode_ci

## 2. 表结构详情

### 2.1 users 表

**描述**: 用户表，存储系统用户信息

| 字段名 | 数据类型 | 约束 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | `int(0)` | `NOT NULL AUTO_INCREMENT` | - | 用户ID |
| `username` | `varchar(50)` | `NOT NULL` | - | 用户名 |
| `password` | `varchar(100)` | `NOT NULL` | - | 密码（加密存储） |
| `role_id` | `int(0)` | `NOT NULL` | - | 角色ID |
| `status` | `enum('active','inactive')` | `NULL` | `'active'` | 用户状态 |
| `created_at` | `datetime(0)` | `NULL` | `CURRENT_TIMESTAMP(0)` | 创建时间 |
| `updated_at` | `datetime(0)` | `NULL` | `CURRENT_TIMESTAMP(0) ON UPDATE CURRENT_TIMESTAMP(0)` | 更新时间 |

**索引**: 
- 主键: `id`
- 唯一索引: `username`
- 普通索引: `idx_users_role_id` (`role_id`)

**引擎**: InnoDB

## 5. 数据库设计说明

1. **用户权限系统**: 采用 RBAC (Role-Based Access Control) 模型，通过 `roles`、`permissions` 和 `role_permissions` 表实现权限管理。

2. **数据关系**: 
   - 用户与角色是多对一关系
   - 角色与权限是多对多关系
   - 商家与商品是一对多关系
   - 用户与订单是一对多关系
   - 商家与订单是一对多关系
   - 订单与物流是一对一关系

3. **索引设计**: 为外键字段和常用查询字段创建了索引，提高查询性能。

4. **数据完整性**: 由应用代码控制数据的完整性和一致性，不再依赖数据库外键约束。

5. **时间戳**: 所有表都包含 `created_at` 和 `updated_at` 字段，记录数据的创建和更新时间。

6. **状态管理**: 对于需要状态管理的表（如用户、产品、活动），使用 `enum` 类型定义状态选项。

## 6. 注意事项

1. **密码安全**: 用户密码使用 bcrypt 算法加密存储，确保安全性。

2. **外键约束**: 已移除所有外键约束，数据完整性由应用代码控制，提高开发灵活性。

3. **字符集**: 使用 `utf8mb4` 字符集，支持表情符号等特殊字符。

4. **数据类型**: 合理选择数据类型，如使用 `decimal` 存储金额，确保精度。

5. **索引优化**: 为常用查询字段创建索引，提高查询性能。
