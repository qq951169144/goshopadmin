## 目录说明

### public/
- **用途**: 存放静态资源，不会被Vite处理
- **favicon.ico**: 网站图标
- **index.html**: HTML模板文件，Vite会自动注入打包后的资源

### src/api/
- **用途**: 定义所有API请求
- **auth.js**: 认证相关API（登录、登出等）
- **merchant.js**: 商户相关API（入驻、信息管理等）
- **product.js**: 商品相关API（发布、管理等）
- **order.js**: 订单相关API（创建、查询等）
- **activity.js**: 活动相关API（配置、管理等）

### src/assets/
- **用途**: 存放项目资源文件
- **images/**: 图片资源
- **styles/**: 样式文件
- **icons/**: 图标资源

### src/components/
- **用途**: 存放通用组件
- **layout/**: 布局组件（如Header、Sidebar、Footer）
- **auth/**: 认证相关组件（如登录表单、权限管理）
- **merchant/**: 商户相关组件（如商户信息表单）
- **product/**: 商品相关组件（如商品列表、商品表单）
- **order/**: 订单相关组件（如订单列表、订单详情）
- **activity/**: 活动相关组件（如秒杀活动配置）

### src/views/
- **用途**: 存放页面视图
- **auth/**: 认证相关页面（如登录页、注册页）
- **merchant/**: 商户相关页面（如商户管理、入驻审核）
- **product/**: 商品相关页面（如商品管理、分类管理）
- **order/**: 订单相关页面（如订单管理、物流跟踪）
- **activity/**: 活动相关页面（如活动管理、秒杀设置）

### src/router/
- **用途**: 配置应用路由
- **index.js**: 定义所有路由，包括路由守卫

### src/store/
- **用途**: 状态管理
- **modules/**: 状态模块（按功能划分）
- **index.js**: 状态管理入口，集成所有模块

### src/utils/
- **用途**: 提供通用工具函数
- **request.js**: HTTP请求工具，基于axios封装
- **storage.js**: 存储工具，封装localStorage和sessionStorage
- **validator.js**: 数据验证工具
- **utils.js**: 其他通用工具函数

### src/constants/
- **用途**: 定义常量
- **api.js**: API相关常量（如API路径）
- **config.js**: 配置相关常量

### src/App.vue
- **用途**: 应用根组件
- **功能**: 包含路由视图，全局布局

### src/main.js
- **用途**: 应用入口
- **功能**: 初始化Vue应用，注册插件，挂载到DOM

## 代码规范

1. **命名规范**:
   - 组件: 大驼峰命名法（如LoginForm.vue）
   - 变量: 小驼峰命名法（如userName）
   - 常量: 全大写，下划线分隔（如API_BASE_URL）
   - 函数: 小驼峰命名法（如loginUser）
   - 文件: 小写字母，短横线分隔（如login-form.vue）

2. **代码风格**:
   - 使用ES6+语法
   - 缩进: 使用2个空格
   - 每行长度: 不超过80个字符
   - 组件属性: 每行一个属性

3. **组件规范**:
   - 组件名要语义化
   - 组件要单一职责
   - 使用Props传递数据
   - 使用Events触发事件
   - 合理使用Vue的生命周期钩子

4. **注释规范**:
   - 组件要有注释说明
   - 复杂逻辑要有注释
   - API请求要有注释说明

## 开发流程

1. **添加新页面**:
   - 在views/中创建页面组件
   - 在router/index.js中注册路由
   - 在api/中添加相关API请求
   - 在components/中创建所需组件

2. **添加新组件**:
   - 在components/中创建组件
   - 在需要使用的页面中导入和注册组件
   - 测试组件功能

3. **API请求**:
   - 在api/中定义API请求函数
   - 在页面或组件中调用API函数
   - 处理请求结果和错误

4. **状态管理**:
   - 在store/modules/中创建状态模块
   - 在store/index.js中注册模块
   - 在页面或组件中使用store

## 构建与部署

1. **开发环境**:
   ```bash
   npm run dev
   ```

2. **构建生产版本**:
   ```bash
   npm run build
   ```

3. **预览生产构建**:
   ```bash
   npm run preview
   ```

4. **部署**:
   - 将dist/目录部署到Web服务器
   - 配置Nginx反向代理到后端服务

## 技术栈

- **框架**: Vue 3
- **构建工具**: Vite
- **UI组件库**: Element Plus
- **状态管理**: Pinia（Vue 3推荐）
- **路由**: Vue Router
- **HTTP请求**: Axios
- **样式**: SCSS/CSS
- **代码规范**: ESLint + Prettier

## 注意事项

1. **性能优化**:
   - 使用Vue的虚拟DOM
   - 合理使用v-if和v-show
   - 避免不必要的计算属性和监听器
   - 使用异步组件和懒加载

2. **安全性**:
   - 避免在前端存储敏感信息
   - 使用HTTPS
   - 防止XSS攻击
   - 验证用户输入

3. **可维护性**:
   - 遵循代码规范
   - 保持组件小型化和专注
   - 使用注释说明复杂逻辑
   - 模块化代码结构

4. **用户体验**:
   - 响应式设计
   - 加载状态和错误提示
   - 合理的动画和过渡效果
   - 清晰的导航结构

## 常见问题

1. **依赖冲突**:
   - 运行 `npm install` 重新安装依赖
   - 检查package.json中的依赖版本

2. **构建失败**:
   - 检查代码语法错误
   - 检查ESLint错误
   - 检查TypeScript类型错误

3. **API请求失败**:
   - 检查API地址是否正确
   - 检查网络连接
   - 检查后端服务是否运行

4. **组件渲染问题**:
   - 检查组件Props
   - 检查组件生命周期
   - 检查数据绑定

## 最佳实践

1. **组件设计**:
   - 容器组件和展示组件分离
   - 合理使用Vue的组合式API
   - 使用插槽（Slots）提高组件灵活性

2. **状态管理**:
   - 只在必要时使用全局状态
   - 合理划分状态模块
   - 使用actions处理异步操作

3. **API请求**:
   - 统一处理API错误
   - 实现请求拦截器和响应拦截器
   - 处理请求超时和重试

4. **样式管理**:
   - 使用CSS变量或SCSS变量
   - 遵循BEM命名规范
   - 避免使用!important

5. **测试**:
   - 为组件编写单元测试
   - 测试API请求
   - 测试用户交互

通过遵循以上规范和最佳实践，可以构建一个结构清晰、性能优异、易于维护的前端应用。
