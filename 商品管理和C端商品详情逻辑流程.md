# 商品管理功能和C端商城商品详情逻辑流程

## 一、商品管理功能逻辑流程

### 1.1 商品创建流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           商品创建流程                                       │
└─────────────────────────────────────────────────────────────────────────────┘

┌──────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  开始    │────▶│ 选择商品分类  │────▶│ 填写基本信息  │────▶│ 上传商品图片  │
└──────────┘     └──────────────┘     └──────────────┘     └──────────────┘
                                                                  │
                                                                  ▼
┌──────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  完成    │◀────│   创建SKU    │◀────│ 设置规格值   │◀────│  定义规格    │
└──────────┘     └──────────────┘     └──────────────┘     └──────────────┘
```

**详细步骤：**

1. **选择商品分类**
   - 从商户的商品分类中选择（必选）
   - 分类为多级结构，支持一级/二级分类
   - 必须先选择分类才能继续后续步骤

2. **填写基本信息**
   - 输入商品名称（必填）
   - 输入商品描述（简短描述）
   - 编辑商品详情（富文本，支持图文混排）
   - 选择商品状态（active/inactive）

3. **上传商品图片**
   - 支持多张图片上传
   - 设置主图（is\_main=1）
   - 支持图片排序
   - 图片存储到对象存储服务

4. **定义商品规格**（必选）
   - 创建规格名称（如：颜色、尺寸、版本）
   - 至少定义一个规格
   - 设置规格排序
   - 一个商品可定义多个规格

5. **设置规格值**
   - 为每个规格添加具体值（如：红色、蓝色）
   - 每个规格至少有一个规格值
   - 设置规格值排序
   - 可禁用某些规格值

6. **创建SKU**
   - 生成规格组合（笛卡尔积）
   - 为每个SKU设置：
     - SKU编码（唯一）
     - 价格（现价）
     - 原价（用于展示优惠）
     - 库存数量
     - 状态
   - 关联规格值（通过product\_sku\_specs表）

7. **保存商品**
   - 计算商品总库存（所有SKU库存之和）
   - 保存到products表
   - 保存规格、规格值、SKU信息

  "values": [

#### SKU生成算法

```
规格组合算法（笛卡尔积）：

规格1（颜色）: [红色, 蓝色, 黑色]
规格2（尺寸）: [S, M, L]

生成的SKU组合：
1. 红色 + S
2. 红色 + M
3. 红色 + L
4. 蓝色 + S
5. 蓝色 + M
6. 蓝色 + L
7. 黑色 + S
8. 黑色 + M
9. 黑色 + L

SKU编码生成规则：
PRODUCT-{product_id}-{规格值编码组合}
例如：PRODUCT-100-RED-M
```

### 3.2 SKU选择算法

```javascript
// 前端SKU选择逻辑示例

class SKUSelector {
  constructor(specifications, skus) {
    this.specifications = specifications; // 规格列表
    this.skus = skus; // SKU列表
    this.selectedSpecs = {}; // 已选择的规格 {specId: specValueId}
  }

  // 选择规格值
  selectSpec(specId, specValueId) {
    this.selectedSpecs[specId] = specValueId;
    return this.getAvailableOptions();
  }

  // 获取可选的规格值
  getAvailableOptions() {
    const available = {};
    
    // 遍历每个规格
    this.specifications.forEach(spec => {
      available[spec.id] = [];
      
      spec.values.forEach(value => {
        // 模拟选择当前规格值
        const testSelection = {...this.selectedSpecs, [spec.id]: value.id};
        
        // 检查是否存在匹配的SKU且有库存
        const hasMatch = this.skus.some(sku => 
          this.matchSpecs(sku, testSelection) && sku.stock > 0
        );
        
        available[spec.id].push({
          ...value,
          available: hasMatch
        });
      });
    });
    
    return available;
  }

  // 检查SKU是否匹配选择的规格
  matchSpecs(sku, selection) {
    for (let specId in selection) {
      const specValueId = selection[specId];
      const match = sku.specs.some(
        s => s.spec_id == specId && s.spec_value_id == specValueId
      );
      if (!match) return false;
    }
    return true;
  }

  // 获取当前选中的SKU
  getSelectedSKU() {
    return this.skus.find(sku => this.matchSpecs(sku, this.selectedSpecs));
  }
}
```

## 四、关键业务规则

### 4.1 商品管理规则

| 规则项      | 说明                        |
| -------- | ------------------------- |
| 商品名称唯一性  | 同一商户下商品名称不能重复             |
| SKU编码唯一性 | 全局唯一，用于库存管理和订单识别          |
| 规格值禁用    | 禁用后前端不展示，但不影响已下单的订单       |
| SKU删除    | 已关联订单的SKU不允许删除，只能禁用       |
| 库存计算     | 商品总库存 = 所有active状态SKU库存之和 |
| 价格显示     | 优先显示SKU价格，无SKU时显示商品基础价格   |

### 4.2 购物车规则

| 规则项   | 说明                               |
| ----- | -------------------------------- |
| 登录状态  | 未登录时购物车数据存储在session/localStorage |
| 合并购物车 | 登录时将本地购物车数据同步到服务器                |
| 库存校验  | 加入购物车时校验库存，结算时再次校验               |
| 价格锁定  | 购物车中的商品价格随商品实时价格变动               |
| 数量限制  | 单次添加数量不能超过SKU库存                  |

### 4.3 订单规则

| 规则项     | 说明              |
| ------- | --------------- |
| 库存锁定    | 创建订单时锁定SKU库存    |
| 库存释放    | 订单取消或超时未支付时释放库存 |
| 价格快照    | 订单中记录商品购买时的价格   |
| SKU信息快照 | 订单中记录SKU的属性信息   |

## 五、异常处理

### 5.1 常见异常情况

| 异常场景    | 处理方式                |
| ------- | ------------------- |
| 商品下架    | 详情页提示"商品已下架"，隐藏购买按钮 |
| SKU无库存  | 规格选择器禁用无库存的组合       |
| 购物车商品下架 | 购物车中标记为"已下架"，不允许结算  |
| 价格变动    | 结算时提示价格已更新          |
| 规格变更    | 刷新页面重新加载规格数据        |

### 5.2 数据一致性保障

1. **库存扣减**：使用数据库事务，确保订单创建和库存扣减原子性
2. **价格同步**：商品详情页实时获取最新价格
3. **缓存策略**：商品基础信息可缓存，库存信息实时查询
4. **并发控制**：热点商品使用分布式锁防止超卖
