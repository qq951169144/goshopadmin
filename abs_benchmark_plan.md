# GoShopAdmin Shop-Backend 性能压测方案

## 1. 测试目标

测试 `goshopadmin-shop-backend` 容器在不同并发情况下的性能表现，包括：
- 响应时间
- 吞吐量（QPS）
- 错误率
- 系统稳定性

## 2. 测试工具

- **工具**：Apache Benchmark (abs.exe)
- **版本**：Apache HTTP Server Benchmarking Tool
- **运行环境**：Windows 系统

## 3. 测试接口

选择以下关键API接口进行测试：

| 接口路径 | 方法 | 功能描述 | 缓存策略 |
| :--- | :--- | :--- | :--- |
| `/api/products` | `GET` | 获取商品列表 | 多级缓存 + 熔断器 |
| `/api/products/{id}` | `GET` | 获取商品详情 | 布隆过滤器 + 缓存 + 分布式锁 |
| `/api/orders` | `POST` | 创建订单 | 无缓存（写操作） |
| `/api/orders/{orderNo}` | `GET` | 获取订单详情 | 布隆过滤器 + 缓存 |

## 4. 测试环境

- **Docker容器**：goshopadmin-shop-backend
- **容器地址**：http://localhost:8081
- **数据库**：MySQL 8.0
- **缓存**：Redis 7.0
- **网络**：本地网络

## 8. 测试结果分析

### 8.1 预期性能指标

| 并发数 | 商品列表QPS | 商品详情QPS | 订单创建QPS | 订单详情QPS |
| :--- | :--- | :--- | :--- | :--- |
| 100 | > 500 | > 1000 | > 200 | > 800 |
| 200 | > 800 | > 1500 | > 300 | > 1200 |
| 500 | > 1000 | > 2000 | > 400 | > 1500 |

### 8.2 瓶颈分析

1. **CPU瓶颈**：
   - 如果CPU使用率持续高于80%，可能存在计算密集型操作
   - 建议：优化代码逻辑，减少不必要的计算

2. **内存瓶颈**：
   - 如果内存使用持续增长，可能存在内存泄漏
   - 建议：检查内存使用情况，优化缓存策略

3. **IO瓶颈**：
   - 如果磁盘IO或网络IO过高，可能存在IO密集型操作
   - 建议：优化数据库查询，增加缓存命中率

4. **数据库瓶颈**：
   - 如果数据库响应缓慢，可能存在查询优化空间
   - 建议：添加索引，优化SQL语句
